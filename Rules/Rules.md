# Community Rules & Workflow

Welcome to our community repository 👋

This document defines the rules and workflow for all members to ensure consistency, collaboration, and good development practices.

---

## 1. General Principles

- Be respectful and constructive in all communications
- Ask questions when in doubt
- Focus on learning, not perfection
- Help other members when possible
- Keep the repository clean and organized

---

## 2. Git Workflow

We use a simple feature-branch workflow.

### Step 1: Create a branch
Each task must be done in a separate branch:

- feature/navbar
- feature/footer
- feature/contact-form
- fix/mobile-layout

---

### Step 2: Work on your feature
Commit frequently with clear and meaningful messages:


git commit -m "Add responsive navbar structure"
git commit -m "Fix footer spacing on mobile"

### Step 3: Push your Branch

git push origin feature/navbar

### Step 4: Create a Pull Request

When opening a PR:

- Target branch: main
- Write a clear description of changes
- Explain what was added or fixed
- Link related Issues if applicable

## 3: Code Review

- At least one member should review each Pull Request (when possible)
- Be respectful when giving feedback
- Suggest improvements instead of criticizing
- Focus on clarity, readability, and structure

## 4: Repository Organization

- Keep files structured and clean
- Avoid mixing unrelated features in the same commit
- Use meaningful folder and file names

## 5: Porpuse of this Organization

- Practice real-world Git workflows
- Improve frontend development skills (HTML, CSS, JS)
- Learn teamwork and collaboration
- Build a strong portfolio for future opportunities

## Final Note by our creator

Clean code. Clear communication. Consistent practice.

Let’s build, learn, and improve together
